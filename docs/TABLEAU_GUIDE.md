# Tableau Public companion dashboard

This guide walks through building a four-worksheet Tableau dashboard on top of the warehouse, so the project shows the literal Power BI / Tableau capability the JD asks for in addition to the Streamlit app.

The Streamlit app is the *interactive* version; Tableau Public is the *shareable static link* version that recruiters and IR directors can open without installing anything.

## 1. Generate the Tableau-ready CSVs

From the project root, with the pipeline already run:

```bash
python scripts/export_for_tableau.py
```

This writes two flat files in `data/`:

- **`tableau_student_view.csv`** — one row per student. Demographics, academic prep, financial aid, first-year aggregates, retention outcome, model risk score and band, and NLP sentiment aggregates.
- **`tableau_term_view.csv`** — one row per (student, term). For time-series and term-level breakdowns.

## 2. Set up the workbook in Tableau Public

1. Install **Tableau Public** (free) from https://www.tableau.com/products/public/download.
2. Create a free Tableau Public account.
3. Open Tableau Public → **Connect → Text File** → pick `tableau_student_view.csv`.
4. On the Data Source tab, drag `tableau_term_view.csv` next to it and join on `student_id` if you want term-level worksheets (use a Left join from student → term).

## 3. Build four worksheets

### Worksheet 1: "Cohort Retention Trends"
- Columns: `cohort_year` (Discrete)
- Rows: `AVG(returned_year_2)` (rename: "Retention rate") — change to Percent format
- Color: `college`
- Filters: `cohort_year`, `college`
- Chart type: Line or grouped bar

### Worksheet 2: "Risk Band Triage"
- Columns: `risk_band` (sorted Low → Moderate → Elevated → High)
- Rows: `COUNT(student_id)` and on a second axis `AVG(returned_year_2)` as a line
- Color: `risk_band` (Low gold → High maroon)
- Tooltip: `pell_eligible`, `first_generation`
- Adds the operational insight: most students are Low-band, but the High band has the highest miss rate

### Worksheet 3: "Equity Lens"
- Columns: `first_generation` × `pell_eligible` (small multiples)
- Rows: `AVG(returned_year_2)`
- Color: dark-to-light by retention rate
- Reveals the intersectional gap (first-gen + Pell-eligible vs. neither)

### Worksheet 4: "Voice of Student"
- Columns: bin(`mean_sentiment`, 0.1)
- Rows: `COUNT(student_id)` (histogram)
- Reference line: `AVG(returned_year_2)` per sentiment band on a secondary axis
- Tooltip: example program, college
- Demonstrates the text → outcome connection

## 4. Compose the dashboard

1. New Dashboard → Size: "Automatic" or 1280x800.
2. Drag all four worksheets into a 2×2 grid.
3. Add a Title: **"Student Success Analytics — IR Companion Dashboard"**.
4. Add a Text object at the bottom: *"Synthetic data, generated by `etl/generate_data.py`. See [github.com/dhavig/student-success-analytics](https://github.com/dhavig/student-success-analytics) for the source pipeline."*
5. Apply filters to the dashboard (cohort year, college) so reviewers can drill down.

## 5. Publish and link

1. **File → Save to Tableau Public As…** — sign in, name it `Student Success Analytics`.
2. Tableau Public will give you a sharable URL like `https://public.tableau.com/views/StudentSuccessAnalytics/Dashboard`.
3. Add the URL to the project README under "Tableau companion."
4. Add the URL to your resume / LinkedIn / Loyola application — recruiters can preview the work in their browser without cloning anything.

## 6. Refresh after re-running the pipeline

Tableau Public's free tier doesn't auto-refresh. When you regenerate:

```bash
python scripts/export_for_tableau.py
```

then in Tableau Public Desktop: **Data → Refresh** → re-publish.
